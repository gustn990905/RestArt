# MVP Scope

## 1. 문서 목적

이 문서는 RestArt의 MVP 범위와 확장 기능 범위를 구분한다.

RestArt는 웹, 모바일 앱, AI 추천 알고리즘, Unity/AR 시뮬레이션, DB, 전시 리플릿 기능을 포함하는 프로젝트이므로 모든 기능을 한 번에 완성하는 방식보다 단계별 개발 범위를 명확히 나누는 것이 필요하다.

---

## 2. 범위 구분 기준

기능 범위는 다음 기준으로 구분한다.

1. 서비스 핵심 가치에 직접 연결되는가
2. 실제 프로토타입 또는 코드 산출물이 존재하는가
3. 사용자 흐름상 먼저 필요한가
4. 추천 모델 또는 DB와 직접 연결되는가
5. 포트폴리오에서 기술 역량을 보여주기에 적합한가

---

## 3. MVP 범위

MVP는 RestArt의 핵심 가치인 “사용자 공간 기반 작품 추천”을 최소 기능으로 구현하는 범위이다.

### MVP에 포함되는 기능

| 기능           | 설명                                            | 우선순위 |
| -------------- | ----------------------------------------------- | -------- |
| 작품 DB        | 작품 이미지, 작가, 설명, 색상, 감성 데이터 저장 | High     |
| 작가/작품 등록 | 작품 추천을 위한 기본 데이터 입력               | High     |
| 이미지 업로드  | 사용자 공간 이미지 또는 작품 이미지 업로드      | High     |
| 대표 색상 추출 | 이미지에서 RGB와 색상 비율 추출                 | High     |
| 감성 태그      | 작품 또는 색상 기반 감성 정보 저장              | High     |
| 작품 추천      | 공간 이미지와 작품 DB를 기반으로 추천 결과 생성 | High     |
| 작품 목록      | 추천 또는 전체 작품 리스트 확인                 | High     |
| 작품 상세      | 작품 정보, 작가 정보, 가격 확인                 | High     |
| 웹 기본 화면   | 추천 흐름을 보여주는 웹 인터페이스              | High     |

### MVP 목표

MVP 단계의 목표는 사용자가 공간 이미지를 업로드하고, 추천된 작품을 확인할 수 있는 기본 흐름을 구현하는 것이다.

MVP 성공 기준은 다음과 같다.

- 작품 데이터가 DB에 저장되어야 한다.
- 공간 이미지 또는 작품 이미지에서 대표 색상을 추출할 수 있어야 한다.
- 색상 및 감성 기반 추천 결과를 생성할 수 있어야 한다.
- 웹 화면에서 추천 결과를 확인할 수 있어야 한다.
- 작품 상세 페이지로 연결될 수 있어야 한다.

---

## 4. Phase 1 확장 범위

Phase 1은 MVP 이후 사용자 경험을 강화하는 기능이다.

### 포함 기능

| 기능               | 설명                                | 우선순위 |
| ------------------ | ----------------------------------- | -------- |
| AR 설치 시뮬레이션 | 추천 작품을 사용자 공간에 가상 배치 | High     |
| 작가 페이지        | 작가 소개와 등록 작품 목록 제공     | Medium   |
| 사용자 컬렉션      | 관심 작품과 구매 작품 저장          | Medium   |
| 추천 사유 표시     | 왜 이 작품이 추천되었는지 설명      | Medium   |
| 작품 감성 수정     | 작가가 AI 추출 감성을 검수하고 수정 | Medium   |

### Phase 1 목표

사용자가 작품을 추천받는 것에서 끝나지 않고, 실제 공간에 배치해보고 저장할 수 있는 경험을 제공한다.

---

## 5. Phase 2 확장 범위

Phase 2는 거래, 리세일, 전시 경험 확장 기능이다.

### 포함 기능

| 기능          | 설명                                           | 우선순위 |
| ------------- | ---------------------------------------------- | -------- |
| 작품 구매     | 작품 상세에서 구매 흐름 제공                   | Medium   |
| 리세일        | 구매 작품을 다시 거래할 수 있는 구조           | Medium   |
| 모바일 리플릿 | 전시장에서 촬영한 작품 기반 개인화 리플릿 생성 | Medium   |
| 전시 추천     | 취향 분석 기반 추천 전시 제공                  | Medium   |
| SNS 공유      | 리플릿 또는 컬렉션을 SNS로 공유                | Medium   |

### Phase 2 목표

RestArt를 단순 추천 서비스가 아니라 작품 거래, 전시 경험, 커뮤니티 홍보까지 연결되는 플랫폼으로 확장한다.

---

## 6. Phase 3 장기 확장 범위

Phase 3는 사업화와 확장성을 고려한 장기 기능이다.

### 포함 기능

| 기능                      | 설명                                | 우선순위 |
| ------------------------- | ----------------------------------- | -------- |
| B2B/B2G 작품 렌탈         | 기업과 기관 대상 작품 렌탈 서비스   | Later    |
| 정기 큐레이션             | 공간 특성에 맞는 작품 정기 교체     | Later    |
| 오프라인 아트마켓         | 예술인 참여형 오프라인 마켓 연계    | Later    |
| PB 상품                   | 리플릿 또는 작품 기반 굿즈 확장     | Later    |
| 글로벌 서비스             | 영어 버전 및 글로벌 전시·작품 추천  | Later    |
| 데이터 기반 운영 대시보드 | 추천, 거래, 사용자 행동 데이터 분석 | Later    |

---

## 7. MVP 제외 범위

MVP에서는 다음 기능을 제외한다.

- 완전한 결제 시스템
- 정식 리세일 정산 시스템
- 기업 대상 렌탈 계약 관리
- 글로벌 서비스
- PB 상품 판매
- 완성형 관리자 대시보드
- 대규모 추천 모델 학습 자동화
- 전체 모바일 앱 정식 출시 기능

이 기능들은 MVP 이후 단계에서 추가한다.

---

## 8. MVP 개발 산출물

MVP 단계에서 필요한 산출물은 다음과 같다.

### 문서

- 제품 요구사항 문서
- 기능 요구사항 문서
- DB 스키마 문서
- API 명세 문서
- 추천 알고리즘 문서

### 코드

- 백엔드 API 프로토타입
- DB schema
- 작품 데이터 seed
- 색상 추출 코드
- 추천 알고리즘 코드
- 웹 프론트 기본 화면

### 디자인

- 추천 흐름 화면
- 작품 목록 화면
- 작품 상세 화면
- 공간 업로드 화면

## 9. MVP 이후 개발 순서

MVP 이후에는 다음 순서로 확장한다.

1. AR 설치 시뮬레이션
2. 작가 페이지
3. 사용자 컬렉션
4. 작품 거래
5. 리세일
6. 모바일 리플릿

8. 모바일 앱
9. B2B/B2G 렌탈
10. 글로벌 서비스

---

## 10. 정리

RestArt의 MVP는 사용자 공간 기반 작품 추천을 중심으로 한다.

이후 AR, 거래, 리플릿, 전시 추천, 렌탈 모델로 확장하여 AI·AR 아트테크 플랫폼의 전체 구조를 완성한다.
