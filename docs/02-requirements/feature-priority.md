# Feature Priority

## 1. 문서 목적

이 문서는 RestArt의 기능 우선순위를 정리한다.

기능 우선순위는 MVP 개발, 기술 문서 작성, 코드 정리, 포트폴리오 구성 순서를 결정하는 기준으로 사용한다.

---

## 2. 우선순위 기준

기능 우선순위는 다음 기준으로 판단한다.

1. 서비스 핵심 가치와 직접 연결되는가
2. 사용자 경험에서 필수적인가
3. AI/AR 아트테크 플랫폼의 차별성을 보여주는가
4. 실제 개발 산출물과 연결되는가
5. 포트폴리오에서 기술 역량을 보여줄 수 있는가

---

## 3. Must Have

Must Have는 RestArt의 핵심 기능을 구성하는 필수 기능이다.

| 기능                      | 이유                                   |
| ------------------------- | -------------------------------------- |
| 작품 DB                   | 모든 추천과 작품 상세의 기반 데이터    |
| 작품 이미지 업로드        | 작품 등록 및 추천 데이터 생성의 시작점 |
| 사용자 공간 이미지 업로드 | 공간 기반 추천의 핵심 입력값           |
| 대표 색상 추출            | 추천 알고리즘의 핵심 데이터            |
| 감성 태그 생성            | 색상 기반 추천의 한계를 보완하는 기준  |
| 작품 추천                 | RestArt의 핵심 기능                    |
| 작품 목록/상세            | 추천 결과를 확인하는 기본 화면         |
| 백엔드 API                | 웹, 앱, DB, AI 모델 연결               |
| DB 스키마                 | 작품, 사용자, 추천 결과 저장 구조      |

---

## 4. Should Have

Should Have는 MVP 이후 사용자 경험을 강화하는 기능이다.

| 기능                    | 이유                          |
| ----------------------- | ----------------------------- |
| AR 작품 설치 시뮬레이션 | 구매 전 불확실성 감소         |
| 작가 페이지             | 예술인 노출 확대              |
| 사용자 컬렉션           | 작품 저장 및 취향 데이터 축적 |
| 추천 사유 표시          | 추천 결과 신뢰도 향상         |
| 작가 감성 검수          | 작가 의도와 AI 분석 차이 보정 |
| 모바일 화면             | AR 및 리플릿 기능과 연결      |

8. 작품 상세 화면
9. Unity/AR 설치 시뮬레이션
10. 모바일 앱 화면
11. 리플릿 기능
12. 디자인 시스템
13. 검증 자료
14. README

## 8. 포트폴리오 강조 기능

포트폴리오에서 강조할 기능은 다음과 같다.

1. AI 색상·감성 기반 작품 추천
2. 이미지 기반 작품 DB 생성
3. 공간 이미지 기반 추천 흐름
4. Unity/AR 작품 설치 시뮬레이션
5. 전시 경험 기반 개인화 모바일 리플릿
6. 웹·모바일·백엔드·AI·DB를 연결한 전체 구조

---

## 9. 정리

RestArt의 우선순위는 작품 추천을 중심으로 한다.

작품 DB, 색상 추출, 감성 분석, 추천 결과 생성이 가장 중요하며, AR 설치 시뮬레이션과 모바일 리플릿은 차별성과 확장성을 보여주는 기능으로 정리한다.
