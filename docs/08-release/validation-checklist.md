# RestArt 검증 체크리스트

이 문서는 RestArt 저장소의 최종 정리 상태를 점검하기 위한 검증 체크리스트이다.

16단계의 목적은 지금까지 정리한 문서, 소스 코드, 프로토타입, 실험 자료가 외부 공개 가능한 상태인지 확인하고, 저장소 릴리즈 기준을 정리하는 것이다.

## 1. 저장소 구조 점검

| 점검 항목                    | 상태 | 비고                                           |
| ---------------------------- | ---- | ---------------------------------------------- |
| 루트 `README.md` 존재        | 완료 | 프로젝트 개요와 핵심 기능 설명 포함            |
| `docs/README.md` 존재        | 완료 | 문서 폴더 전체 목차 정리                       |
| `apps/README.md` 존재        | 완료 | 애플리케이션 및 프로토타입 구조 정리           |
| `tools/README.md` 존재       | 완료 | 추천 모델, 이미지 매칭 도구 구조 정리          |
| `experiments/README.md` 존재 | 완료 | 색상 분석 및 멀티모달 감성 분석 실험 구조 정리 |
| `design/README.md` 존재      | 완료 | 디자인 자료 관리 기준 정리                     |
| `assets/README.md` 존재      | 완료 | 정적 자산 관리 기준 정리                       |

## 2. 불필요한 placeholder 폴더 정리

실제 소스, 문서, 공개 가능한 자산이 없는 빈 구조 폴더는 삭제하였다.

| 삭제된 항목                | 삭제 이유                                            |
| -------------------------- | ---------------------------------------------------- |
| `infra/`                   | 현재 배포 및 인프라 설정 파일을 포함하지 않음        |
| `packages/`                | 현재 공통 패키지 또는 UI 패키지를 별도 관리하지 않음 |
| `references/`              | 공개 가능한 참고 원본 자료를 포함하지 않음           |
| `assets/images/`           | 공개 이미지 자산을 포함하지 않음                     |
| `assets/mockups/`          | 공개 mockup 자산을 포함하지 않음                     |
| `assets/videos/`           | 공개 영상 자산을 포함하지 않음                       |
| `design/figma/`            | 공개 가능한 디자인 원본 파일을 포함하지 않음         |
| `design/prototype-frames/` | 공개 가능한 prototype frame을 포함하지 않음          |
| `design/screen-captures/`  | 공개 가능한 화면 캡처 자료를 포함하지 않음           |

## 5. 민감 정보 점검

저장소에는 실제 비공개 접속 정보나 개인 자료가 포함되지 않아야 한다.

| 점검 항목                                  | 상태 |
| ------------------------------------------ | ---- |
| 실제 DB 접속값 하드코딩 없음               | 확인 |
| 실제 서비스 인증 정보 없음                 | 확인 |
| 개인정보 포함 문서 없음                    | 확인 |
| 원본 사업 문서 전체 파일 없음              | 확인 |
| 대용량 비공개 이미지 또는 영상 결과물 없음 | 확인 |
| 로컬 개발 환경 폴더 없음                   | 확인 |
| IDE 설정 폴더 없음                         | 확인 |
| 자동 생성 cache 폴더 없음                  | 확인 |

## 6. 알려진 제한 사항

일부 Python 소스는 `config` 또는 `db_config`와 같은 변수명을 통해 DB 접속 설정을 참조한다.

이는 실제 비공개 값이 저장소에 포함된 것이 아니라, 실행 시 별도 로컬 설정을 통해 값을 주입하는 구조이므로 공개 저장소 기준에서는 허용 가능하다.

다만 실제 실행을 위해서는 다음 항목이 별도로 필요하다.

- 로컬 DB 설정
- 실행 환경 설정
- 필요한 Python package 설치
- sample data 또는 테스트용 데이터 구성

## 7. 실행 검증 상태

| 영역                      | 상태 | 비고                                                |
| ------------------------- | ---- | --------------------------------------------------- |
| Python 문법 검사          | 보류 | 현재 로컬 Python launcher가 정상 동작하지 않는 상태 |
| Backend API 실행          | 보류 | 의존성 설치와 DB 설정 필요                          |
| FastAPI web 실행          | 보류 | 의존성 설치와 경로 설정 필요                        |
| Static web prototype 확인 | 가능 | HTML/CSS 파일 직접 확인 가능                        |
| Markdown 문서 확인        | 가능 | 저장소 내 문서로 확인 가능                          |

## 8. 릴리즈 준비 상태 요약

현재 RestArt 저장소는 공개 가능한 프로젝트 정리 저장소이자 개발 prototype 저장소로 정리되어 있다.

현재 저장소는 다음 목적에 적합하다.

- 프로젝트 개념 설명
- 기능 구조 검토
- Backend 및 web prototype 검토
- 추천 logic 검토
- 이미지·색상 분석 실험 검토
- 문서 기반 프로젝트 설명

현재 저장소는 production 배포 패키지는 아니다.

production 배포를 위해서는 다음 작업이 추가로 필요하다.

- 실행 환경 고정
- 의존성 version 고정
- 실제 실행 테스트
- 배포 설정 작성
- DB migration 또는 초기화 절차 작성
- sample data 구성
- API 실행 문서 보완
