# 감성 및 색상 테이블 요약

이 문서는 RestArt prototype에서 사용한 감성 정보와 색상 정보의 데이터 구조를 요약한다.

RestArt의 작품 추천은 단순히 이미지 유사도만 비교하는 방식이 아니라, 작품의 색상 정보와 감성 정보를 함께 활용하여 사용자 이미지, 공간 이미지, 전시 이미지와 어울리는 작품을 추천하는 구조를 가진다.

## 기능 목적

감성 및 색상 테이블의 목적은 작품 데이터를 추천 가능한 형태로 변환하는 것이다.

작품은 이미지 URL, 제목, 작가, 설명, 색상 정보, 감성 정보와 연결될 수 있으며, 이 정보는 다음 기능에서 사용된다.

- 공간 기반 작품 추천
- 모바일 리플릿 생성
- 색상 기반 작품 추천
- 감성 기반 작품 추천
- 전시 추천 결과와 작품 추천 결과 연결
- 사용자 취향 설명

## 작품 데이터의 주요 구성

RestArt의 추천 로직에서 작품 데이터는 다음 정보와 연결될 수 있다.

- 작품 이미지 URL
- 작품 제목
- 작가명
- 작품 설명
- 대표 색상
- 색상 cluster 정보
- 색상 비율
- 감성 태그
- 추천 기준값

이 정보는 사용자 입력 이미지 또는 공간 이미지의 분석 결과와 비교되며, 추천 작품 후보를 선정하는 기준이 된다.

## 색상 정보의 역할

색상 정보는 작품 이미지와 사용자 이미지 사이의 시각적 유사성을 판단하기 위해 사용된다.

색상 정보는 다음과 같은 형태로 활용될 수 있다.

- 작품 이미지의 대표 색상
- 작품 이미지의 색상 cluster
- 색상 cluster 비율
- RestArt 색상 dictionary와의 매칭 결과
- 색상 이름 또는 색상 그룹

색상 정보의 주요 역할은 다음과 같다.

1. 사용자 이미지와 시각적으로 어울리는 작품 탐색
2. 공간 이미지와 작품 이미지의 색상 연결
3. 모바일 리플릿의 대표 색상 및 디자인 타입 결정
4. 추천 결과에 색상 설명 제공
5. 감성 추천 결과를 보완하는 시각적 기준 제공

## 감성 정보의 역할

감성 정보는 작품이 전달하는 분위기와 사용자의 이미지 또는 공간 이미지가 가진 분위기를 연결하기 위해 사용된다.

감성 정보는 다음과 같은 형태로 활용될 수 있다.

- 작품별 감성 태그
- 작품 설명에서 도출된 분위기
- 이미지 기반 감성 예측 결과
- 사용자 입력 이미지의 감성 또는 분위기
- 공간 이미지의 감성적 방향

감성 정보의 주요 역할은 다음과 같다.

1. 색상만으로 설명하기 어려운 작품 분위기 반영
2. 사용자 이미지의 정서적 특성과 어울리는 작품 추천
3. 공간 기반 작품 추천에서 분위기 적합성 판단
4. 모바일 리플릿 결과에서 감성 기반 추천 작품 구성
5. 작품 추천 결과의 설명 가능성 강화

## 색상 정보와 감성 정보의 결합

RestArt의 추천 구조에서는 색상 정보와 감성 정보를 분리해서만 사용하지 않고, 두 정보를 함께 사용할 수 있다.

색상 정보는 이미지의 시각적 특성을 설명하고, 감성 정보는 이미지와 작품이 가진 분위기를 설명한다.

두 정보를 결합하면 다음과 같은 추천 구조를 만들 수 있다.

1. 사용자 이미지 또는 공간 이미지 입력
2. 대표 색상 추출
3. RestArt 색상 dictionary와 매칭
4. 이미지 또는 텍스트 기반 감성 정보 도출
5. 작품 데이터의 색상 정보와 비교
6. 작품 데이터의 감성 태그와 비교
7. 색상 및 감성 기준을 함께 반영한 작품 추천

## 모바일 리플릿 기능에서의 사용

모바일 리플릿 생성 기능에서는 사용자가 입력한 전시장 또는 작품 이미지와 관련된 추천 결과를 구성한다.

이때 색상 정보와 감성 정보는 다음 방식으로 사용된다.

- 사용자 이미지에서 대표 색상 추출
- 대표 색상과 작품 색상 정보 비교
- 색상 기반 추천 작품 선정
- 감성 기반 추천 작품 선정
- 추천 전시 정보와 연결
- 리플릿 디자인 타입 선택

이 구조를 통해 모바일 리플릿 결과는 단순 이미지 매칭 결과가 아니라, 색상과 감성 기준이 함께 반영된 개인화 결과가 된다.

## 공간 기반 작품 추천 기능에서의 사용

공간 기반 작품 추천 기능에서는 사용자가 입력한 공간, 인테리어, 장소 이미지를 분석한다.

이때 색상 정보와 감성 정보는 다음 방식으로 사용된다.

- 공간 이미지의 대표 색상 추출
- 공간 이미지의 분위기 또는 감성 정보 분석
- 작품 데이터의 색상 정보와 비교
- 작품 데이터의 감성 태그와 비교
- 공간과 어울리는 작품 후보 선정

이 구조를 통해 사용자는 자신의 공간 분위기와 어울리는 작품을 추천받을 수 있다.

## 데이터 공개 범위

이 repository에는 전체 원본 감성 데이터, 내부 테이블 자료, 비공개 기획 문서를 그대로 포함하지 않는다.

대신 공개 가능한 범위에서 다음 정보만 정리한다.

- 감성 및 색상 정보의 역할
- 추천 로직에서의 사용 방식
- 데이터 구조의 요약
- 색상 dictionary 코드
- 추천 pipeline과의 연결 관계

## 정리 기준

이 문서는 기존 prototype과 내부 자료를 기반으로 하되, 공개 가능한 형태로 재정리한 요약 문서다.

정리 기준은 다음과 같다.

- 원본 데이터 전체를 직접 포함하지 않음
- 민감한 내부 문서나 비공개 자료 제외
- 작품 추천 로직에 필요한 구조 중심으로 설명
- 색상 정보와 감성 정보의 기능적 역할을 분리해서 정리
- 모바일 리플릿, 공간 기반 작품 추천, 색상 분석 문서와 연결
