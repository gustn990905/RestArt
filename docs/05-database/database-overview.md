# Database Overview

## 개요

이 문서는 RestArt 프로젝트의 데이터베이스 전체 구조를 설명한다.

RestArt는 사용자의 공간 이미지와 작품 데이터를 분석하여 어울리는 작품을 추천하고, 전시장에서 촬영한 작품 이미지를 기반으로 개인화 모바일 리플릿을 생성하는 AI·AR 기반 아트테크 플랫폼이다.

따라서 데이터베이스는 단순히 작품 목록만 저장하는 구조가 아니라 다음 기능을 함께 지원해야 한다.

- 작가 및 작품 데이터 관리
- 작품 색상 정보 저장
- 작품 감성 정보 저장
- 사용자 공간 이미지 분석 결과 저장
- 공간 기반 작품 추천 결과 저장
- 전시 정보 저장
- 모바일 리플릿 생성 결과 저장
- 색상 기준 테이블 관리
- 감성 기준 테이블 관리
- 전문가 인테리어 및 기준 작품 데이터 관리

---

## DB 설계 기준

RestArt의 DB는 다음 세 가지 흐름을 기준으로 설계한다.

### 1. 작품 DB 생성 흐름

작가가 작품을 등록하면 작품 이미지, 작품명, 작가명, 작품 설명을 기반으로 감성 추출 알고리즘과 RGB 추출 알고리즘을 실행한다.

이후 추출된 결과를 작품 DB에 저장한다.

저장되는 주요 정보는 다음과 같다.

- 작가 정보
- 작품 이미지 URL
- 작품명
- 작품 설명
- 감성 태그
- 대표 색상
- 색상 RGB
- 색상 비율
- 색상 클러스터 정보

이 데이터는 RestArt 추천 시스템의 핵심 기반 데이터가 된다.

---

### 2. 공간 이미지 기반 작품 추천 흐름

사용자는 작품을 설치하고 싶은 공간 사진을 업로드한다.

시스템은 사용자 공간 이미지에서 대표 색상과 색상 비율을 추출하고, 이를 전문가 인테리어 DB 및 RestArt 작품 DB와 비교한다.

추천 흐름은 다음과 같다.

1. 사용자 공간 이미지 업로드
2. K-means clustering 기반 대표 색상 추출
3. 전문가 인테리어 DB와 유사도 비교
4. 전문가 인테리어와 연결된 기준 작품 탐색
5. 기준 작품과 RestArt 작품 DB 비교
6. 감성 일치 개수 기준 정렬
7. 켄달타우 점수 기준 정렬
8. 추천 작품 목록 생성
9. 추천 결과 저장

이 흐름을 위해 DB는 사용자 공간 이미지 분석 결과와 추천 결과를 저장할 수 있어야 한다.

---

### 3. 전시 경험 기반 모바일 리플릿 흐름

사용자는 전시장에서 마음에 드는 작품을 촬영한다.

촬영된 이미지는 전시 작품 DB 또는 RestArt 작품 DB와 비교되며, 사용자가 촬영한 이미지들을 기반으로 개인화 리플릿을 생성한다.

리플릿 생성 흐름은 다음과 같다.

1. 전시장 작품 촬영
2. 촬영 이미지 저장
3. 촬영 이미지와 전시 작품 DB 비교
4. 정형화된 작품 데이터로 매핑
5. 대표 RGB 또는 대대표 색상 계산
6. 대표 이미지 또는 대표 작품 선정
7. 색상 성격 및 감성 텍스트 생성
8. 추천 작품 생성
9. 추천 전시 생성
10. 모바일 리플릿 결과 저장

이 흐름을 위해 DB는 리플릿 세션, 촬영 이미지, 리플릿 추천 결과, 전시 정보를 저장할 수 있어야 한다.

---

## 통합 DB 구조

RestArt는 하나의 통합 플랫폼으로 정리한다.

공간 기반 작품 추천 기능과 모바일 리플릿 기능을 별도 프로젝트로 분리하지 않고, 하나의 DB 구조 안에서 기능별 테이블을 분리한다.

전체 DB는 다음 그룹으로 구성한다.

| 그룹                  | 설명                                                   |
| --------------------- | ------------------------------------------------------ |
| Core Data             | 사용자, 작가, 작품 기본 정보                           |
| Artwork Analysis Data | 작품 색상, 작품 감성, 작품 설명 분석 결과              |
| Recommendation Data   | 사용자 공간 이미지 분석, 추천 결과                     |
| Leaflet Data          | 전시 촬영 이미지, 리플릿 생성 결과                     |
| Exhibition Data       | 전시 정보, 전시 기관, 전시 위치                        |
| Reference Data        | 감성 기준 테이블, 색상 기준 테이블, 전문가 인테리어 DB |

---

## 주요 테이블 그룹

### 1. Core Data

RestArt 플랫폼의 기본 데이터를 저장한다.

- `users`
- `artists`
- `artworks`

### 2. Artwork Analysis Data

작품 분석 결과를 저장한다.

- `artwork_colors`
- `artwork_emotions`
- `color_dictionary`
- `emotion_dictionary`

### 3. Recommendation Data

공간 이미지 기반 작품 추천 결과를 저장한다.

- `user_space_images`
- `recommendation_results`
- `expert_interiors`
- `expert_artworks`

### 4. Leaflet Data

전시 경험 기반 모바일 리플릿 생성 결과를 저장한다.

- `leaflet_sessions`
- `leaflet_images`
- `leaflet_recommendations`

### 5. Exhibition Data

전시 정보와 전시 추천 기능을 위한 데이터를 저장한다.

- `exhibitions`

## 기존 코드와의 관계

Day 4에서 정리한 DB loader 코드는 기존 `images` 테이블에 작품 데이터를 삽입하는 구조를 가지고 있었다.

기존 삽입 대상 컬럼은 다음과 같다.

- `url`
- `title`
- `author`
- `description`
- `emotions`
- `cluster_rgb`
- `cluster_ratio`
- `cluster_count`
- `main_color`

Day 5에서는 이 구조를 기반으로 하되, 포트폴리오와 유지보수성을 고려하여 DB 구조를 다음과 같이 분리한다.

| 기존 구조              | 개선 구조                                  |
| ---------------------- | ------------------------------------------ |
| `images.url`           | `artworks.image_url`                       |
| `images.title`         | `artworks.title`                           |
| `images.author`        | `artists.name` 또는 `artworks.artist_name` |
| `images.description`   | `artworks.description`                     |
| `images.emotions`      | `artwork_emotions`                         |
| `images.cluster_rgb`   | `artwork_colors.rgb_value`                 |
| `images.cluster_ratio` | `artwork_colors.ratio`                     |
| `images.cluster_count` | `artwork_colors.cluster_count`             |
| `images.main_color`    | `artworks.main_color`                      |

즉, 기존 코드를 부정하는 것이 아니라, 기존 구현 구조를 더 명확한 DB 설계로 확장한다.

## 색상 데이터 설계 방향

RestArt는 K-means clustering 기반 RGB 추출 알고리즘을 사용한다.

작품 이미지 또는 사용자 공간 이미지에서 대표 RGB와 색상 비율을 추출하고, 이를 추천 알고리즘에 활용한다.

색상 데이터는 다음 항목을 기준으로 저장한다.

- 색상명
- RGB 값
- 색상 비율
- 클러스터 순서
- 클러스터 count
- 대표 색상 여부

색상 기준 테이블은 RestArt 자체 색상 테이블 또는 Hue & Tone System 기반 색상 매핑 구조를 참고하여 구성한다.

---

## 감성 데이터 설계 방향

RestArt는 예술 작품 감상 시 느끼는 감성을 자체 감성 테이블로 정리한다.

감성 데이터는 작품 이미지, 작품명, 작품 설명을 기반으로 추출되며, RestArt 감성 카테고리에 매핑된다.

감성 데이터는 다음 항목을 기준으로 저장한다.

- 감성명
- 감성 카테고리
- 감성 설명
- 작품별 감성 순위
- 작가 수정 여부
- AI 추출 여부

작가가 AI가 추출한 감성을 수정할 수 있도록 설계하여, 작가 의도와 추천 데이터 품질을 함께 반영한다.

---

## 추천 데이터 설계 방향

RestArt 추천 알고리즘은 다음 두 가지 기준을 함께 사용한다.

1. 감성 일치 개수
2. 켄달타우 점수

추천 결과는 단순히 색상이 비슷한 작품을 보여주는 것이 아니라, 사용자 공간 또는 촬영 이미지와 작품 DB 사이의 감성·색상 관계를 함께 고려한다.

추천 결과 테이블은 다음 정보를 저장한다.

- 사용자 ID
- 업로드 이미지 ID
- 추천 작품 ID
- 감성 일치 개수
- 켄달타우 점수
- 추천 순위
- 추천 사유
- 추천 생성일

---

## 리플릿 데이터 설계 방향

모바일 리플릿 기능은 전시장에서 촬영한 작품 이미지를 기반으로 한다.

리플릿 관련 DB는 다음 데이터를 저장한다.

- 리플릿 세션
- 사용자가 촬영한 이미지
- 매핑된 작품
- 대표 색상
- 색상 성격 텍스트
- 추천 작품
- 추천 전시
- SNS 공유용 데이터

이 구조를 통해 전시 경험을 단순 관람에서 개인화된 디지털 콘텐츠로 확장한다.

---

## DB 설계 요약

RestArt DB는 다음 방향으로 설계한다.

1. 작품 DB를 플랫폼의 핵심 데이터로 둔다.
2. 작품 색상과 감성은 별도 테이블로 분리한다.
3. 사용자 공간 이미지 분석 결과를 저장한다.
4. 추천 결과는 재조회 가능하도록 저장한다.
5. 리플릿 생성 과정과 결과를 별도 테이블로 관리한다.
6. 색상 기준 테이블과 감성 기준 테이블을 reference data로 둔다.
7. 기존 `images` 중심 loader 구조를 실무형 테이블 구조로 확장한다.

## 향후 개선 방향

향후 DB는 다음 방향으로 개선할 수 있다.

- ERD 이미지 작성
- 작품 색상 테이블 정규화
- 작품 감성 테이블 정규화
- 추천 결과 로그 테이블 추가
- 사용자 행동 로그 테이블 추가
- 리플릿 공유 이력 테이블 추가
- 관리자 검수 테이블 추가
- 실제 운영 DB migration 구조 도입
