# Leaflet Recommendation Logic

## 개요

이 문서는 RestArt의 전시 경험 기반 모바일 리플릿 추천 로직을 정리한다.

모바일 리플릿 기능은 사용자가 전시장에서 마음에 드는 작품을 촬영하면, 촬영 이미지들을 기반으로 사용자의 색상 취향과 감성 취향을 분석하고 개인화된 리플릿 결과를 생성하는 기능이다.

이 기능은 단순한 전시 안내문이 아니라, 사용자의 관람 경험을 데이터화하여 작품 추천과 전시 추천으로 확장하는 구조를 가진다.

---

## 모바일 리플릿의 목적

모바일 리플릿 기능의 목적은 다음과 같다.

1. 사용자가 촬영한 작품을 기반으로 개인화된 전시 기록을 생성한다.
2. 사용자의 시각적 취향을 대표 색상으로 해석한다.
3. 촬영 작품의 감성을 기반으로 취향을 분석한다.
4. 사용자에게 어울리는 작품을 추가 추천한다.
5. 사용자에게 관련 전시를 추천한다.
6. 전시 관람 경험을 SNS 공유 가능한 콘텐츠로 전환한다.

---

## 입력 데이터

모바일 리플릿 기능은 다음 데이터를 입력값으로 사용한다.

| 입력 데이터      | 설명                                 |
| ---------------- | ------------------------------------ |
| 촬영 작품 이미지 | 사용자가 전시장에서 촬영한 작품 사진 |
| 전시 정보        | 전시명, 기관, 위치, 포스터, 설명     |
| 작품 DB          | 촬영 작품과 매칭할 기준 작품 데이터  |
| 작품 색상 데이터 | 작품별 RGB, 색상 비율, 대표 색상     |
| 작품 감성 데이터 | 작품별 감성 태그                     |
| 색상 기준 테이블 | 색상명, 대표 RGB, 성격 문구          |
| 추천 후보 작품   | 리플릿 결과에서 추천할 작품 목록     |

---

## 전체 처리 흐름

```text
전시장 작품 촬영
→ 촬영 이미지 업로드
→ 리플릿 세션 생성
→ 촬영 이미지와 작품 DB 매칭
→ 촬영 작품의 색상 및 감성 데이터 조회
→ 대표 색상 계산
→ 대표 작품 선정
→ 색상 성격 문구 생성
→ 추천 작품 생성
→ 추천 전시 생성
→ 모바일 리플릿 결과 저장
```

---

## 1. 리플릿 세션 생성

사용자가 리플릿을 만들기 시작하면 하나의 리플릿 세션이 생성된다.

리플릿 세션은 사용자가 특정 전시에서 촬영한 이미지 묶음을 관리하는 단위이다.

### 저장 테이블

```text
leaflet_sessions
```

### 주요 저장 데이터

| 필드                  | 설명                |
| --------------------- | ------------------- |
| `leaflet_session_id`  | 리플릿 세션 ID      |
| `user_id`             | 사용자 ID           |
| `exhibition_id`       | 전시 ID             |
| `dominant_color_name` | 대표 색상명         |
| `dominant_rgb_r`      | 대표 R 값           |
| `dominant_rgb_g`      | 대표 G 값           |
| `dominant_rgb_b`      | 대표 B 값           |
| `personality_text`    | 색상 기반 성격 문구 |

---

## 2. 촬영 이미지 저장

사용자가 촬영한 작품 이미지는 리플릿 세션에 연결되어 저장된다.

### 저장 테이블

```text
leaflet_images
```

### 주요 저장 데이터

| 필드                   | 설명                          |
| ---------------------- | ----------------------------- |
| `leaflet_session_id`   | 연결된 리플릿 세션            |
| `uploaded_image_url`   | 사용자가 업로드한 촬영 이미지 |
| `matched_artwork_id`   | DB에서 매칭된 작품 ID         |
| `selected_for_leaflet` | 대표 이미지 선정 여부         |

---

## 3. 촬영 이미지와 작품 DB 매칭

사용자가 촬영한 작품 이미지는 DB에 저장된 작품 이미지와 비교된다.

이 과정은 사용자가 직접 작품명을 입력하지 않아도 촬영 이미지가 어떤 작품에 가까운지 판단하기 위한 단계이다.

### 매칭 방식 후보

- 이미지 특징점 기반 비교
- 평균 RGB 기반 비교
- 색상 분포 기반 비교
- 유클리드 거리 기반 비교
- AKAZE 등 이미지 매칭 알고리즘 활용 가능

매칭 결과가 생성되면 `leaflet_images.matched_artwork_id`에 저장한다.

---

## 4. 촬영 작품의 색상 데이터 조회

촬영 이미지가 작품 DB와 매칭되면, 해당 작품의 색상 데이터를 조회한다.

### 조회 테이블

- `artworks`
- `artwork_colors`
- `color_dictionary`

### 활용 데이터

- 대표 색상
- RGB 값
- 색상 비율
- 색상 순위
- 색상명
- 색상 성격 문구

---

## 5. 촬영 작품의 감성 데이터 조회

촬영 작품과 매칭된 작품 ID를 기준으로 작품의 감성 데이터를 조회한다.

### 조회 테이블

- `artwork_emotions`
- `emotion_dictionary`

### 활용 데이터

- 감성명
- 감성 순위
- 감성 카테고리
- 감성 설명

이 데이터는 리플릿 추천 작품 선정과 사용자 취향 해석에 활용된다.

---

## 6. 대표 색상 계산

리플릿은 사용자가 촬영한 여러 작품의 색상 정보를 종합하여 대표 색상을 계산한다.

### 계산 방식

```text
촬영 작품들의 RGB 값 수집
→ 평균 RGB 계산
→ 기준 색상 테이블과 비교
→ 가장 가까운 색상명 선택
→ 대표 색상 저장
```

대표 색상은 `leaflet_sessions`에 저장된다.

---

## 7. 색상 성격 문구 생성

대표 색상이 결정되면 `color_dictionary`의 `personality_text`를 활용하여 사용자에게 보여줄 문구를 생성한다.

예시:

```text
차분하면서도 감각적인 분위기를 선호하는 사용자입니다.
```

이 문구는 리플릿 결과 페이지에서 사용자의 취향 설명으로 제공된다.

---

## 8. 대표 작품 선정

리플릿에서는 사용자가 촬영한 작품 중 일부를 대표 이미지로 선정한다.

대표 작품 선정 방식은 다음과 같다.

```text
촬영 이미지 목록 수집
→ 매칭된 작품의 RGB 데이터 조회
→ 평균 RGB 또는 중심 색상 계산
→ 전체 색상 분포와 차이를 보이는 작품 선정
→ 대표 이미지로 표시
```

대표 작품은 `leaflet_images.selected_for_leaflet` 값으로 표시할 수 있다.

---

## 9. 추천 작품 생성

리플릿 결과에는 사용자의 촬영 취향과 어울리는 추천 작품을 포함할 수 있다.

### 추천 기준

- 촬영 작품과 감성이 유사한 작품
- 대표 색상과 색상 구성이 유사한 작품
- 켄달타우 점수가 높은 작품
- 사용자가 촬영한 작품과 다른 작가의 작품
- 리플릿 대표 색상과 잘 맞는 작품

### 추천 흐름

```text
촬영 작품 감성 수집
→ 촬영 작품 색상 수집
→ 추천 후보 작품 조회
→ 감성 일치 개수 계산
→ 켄달타우 점수 계산
→ 추천 작품 정렬
→ 상위 추천 작품 저장
```

### 저장 테이블

```text
leaflet_recommendations
```

---

## 10. 추천 전시 생성

모바일 리플릿은 작품 추천뿐 아니라 전시 추천으로도 확장될 수 있다.

전시 추천은 다음 데이터를 활용한다.

- 전시명
- 전시 기관
- 전시 위치
- 전시 포스터
- 전시 설명
- 사용자 대표 색상
- 사용자 선호 감성
- 촬영 작품과 연결된 작가 또는 장르

### 저장 테이블

- `exhibitions`
- `leaflet_recommendations`

## 11. 리플릿 결과 구성

최종 모바일 리플릿은 다음 항목으로 구성된다.

| 구성 요소        | 설명                             |
| ---------------- | -------------------------------- |
| 대표 색상        | 촬영 작품 기반으로 계산된 색상   |
| 색상 성격 문구   | 대표 색상에 연결된 취향 문구     |
| 촬영 작품 이미지 | 사용자가 선택하거나 촬영한 작품  |
| 대표 작품        | 리플릿에 강조 표시되는 작품      |
| 추천 작품        | 취향 기반 추천 작품              |
| 추천 전시        | 관련 전시 정보                   |
| 공유 콘텐츠      | SNS 공유용 결과 이미지 또는 링크 |

## 12. API와의 연결

모바일 리플릿 기능은 Day 4의 API prototype과 다음 방식으로 연결될 수 있다.

| API 후보                                 | 역할                          |
| ---------------------------------------- | ----------------------------- |
| `POST /leaflet_creating/`                | 리플릿 세션 생성 및 결과 생성 |
| `POST /analyze-most-common-rgb/`         | 촬영 이미지의 대표 RGB 분석   |
| `POST /analyze-most-different-pictures/` | 대표 이미지 선정              |
| `GET /find_near_exhibition/`             | 주변 전시 또는 관련 전시 추천 |

## 정리

RestArt의 모바일 리플릿 추천 로직은 사용자가 전시장에서 촬영한 작품 이미지를 취향 데이터로 변환하는 구조이다.

촬영 이미지는 작품 DB와 매칭되고, 매칭된 작품의 색상과 감성 데이터를 기반으로 대표 색상, 성격 문구, 추천 작품, 추천 전시가 생성된다.

이를 통해 RestArt는 전시 관람 경험을 개인화된 디지털 리플릿과 추천 경험으로 확장한다.
