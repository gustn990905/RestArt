# AI Recommendation Architecture

## 개요

이 문서는 RestArt 프로젝트의 AI 추천 시스템 전체 구조를 정리한다.

RestArt는 사용자의 공간 이미지와 작품 데이터를 분석하여 어울리는 작품을 추천하고, 전시장에서 촬영한 작품 이미지를 기반으로 개인화 모바일 리플릿을 생성하는 AI·AR 기반 아트테크 플랫폼이다.

AI 추천 시스템은 단순히 이미지를 비교하는 기능이 아니라, 작품 이미지, 공간 이미지, 색상 데이터, 감성 데이터, 전시 데이터, 사용자 촬영 데이터를 함께 활용하는 구조로 설계된다.

---

## 추천 시스템의 핵심 목적

RestArt AI 추천 시스템의 목적은 다음과 같다.

1. 사용자의 공간 분위기와 어울리는 작품 추천
2. 작품의 색상과 감성을 기반으로 한 개인화 추천
3. 전시장에서 촬영한 작품을 활용한 모바일 리플릿 생성
4. 사용자의 취향을 시각적 데이터와 감성 데이터로 해석
5. 작가 작품 DB와 전시 DB를 연결한 작품 탐색 경험 제공

---

## 전체 추천 구조

RestArt의 AI 추천 시스템은 다음 네 가지 분석 흐름을 중심으로 구성된다.

```text
작품 DB 생성
→ 색상 추출
→ 감성 매핑
→ 유사도 및 순위 계산
→ 추천 결과 생성
```

추천 시스템은 크게 두 가지 사용자 기능으로 연결된다.

```text
1. 공간 이미지 기반 작품 추천
2. 전시 경험 기반 모바일 리플릿 추천
```

두 기능은 별도 시스템이 아니라, 동일한 작품 DB, 색상 분석 결과, 감성 분석 결과, 추천 정렬 로직을 공유한다.

---

## 전체 아키텍처

```text
[User Input]
    ├─ 공간 이미지 업로드
    └─ 전시장 작품 이미지 촬영

[Image Analysis Layer]
    ├─ RGB 색상 추출
    ├─ 대표 색상 추출
    ├─ 색상 비율 계산
    └─ 촬영 이미지 매칭

[Emotion Analysis Layer]
    ├─ 작품 이미지 분석
    ├─ 작품명 분석
    ├─ 작품 설명 분석
    └─ RestArt 감성 카테고리 매핑

[Recommendation Layer]
    ├─ 감성 일치 개수 계산
    ├─ 켄달타우 점수 계산
    ├─ 유사 작품 탐색
    ├─ 추천 작품 정렬
    └─ 추천 결과 생성

[Service Layer]
    ├─ 공간 기반 작품 추천
    ├─ 모바일 리플릿 생성
    ├─ 전시 추천
    └─ AR 작품 배치 기능과 연결
```

## 1. 작품 DB 생성 단계

작품 DB는 RestArt 추천 시스템의 중심 데이터이다.

작가가 작품을 등록하면 시스템은 작품 이미지, 작품명, 작품 설명을 기반으로 작품 분석 데이터를 생성한다.

### 작품 등록 입력값

- 작가 ID
- 작가명 또는 닉네임
- 작품명
- 작품 이미지 URL
- 작품 설명
- 작품 가격
- 작품 상태

### AI 분석 결과

- 대표 색상
- RGB 색상값
- 색상 비율
- 색상 클러스터 정보
- 감성 태그
- 감성 순위
- 감성 생성 출처

### 연결 DB

- `artists`
- `artworks`
- `artwork_colors`
- `artwork_emotions`
- `color_dictionary`
- `emotion_dictionary`

## 2. 색상 추출 단계

색상 추출은 작품 추천과 리플릿 생성에 모두 사용되는 공통 기능이다.

작품 이미지 또는 공간 이미지를 입력받아 K-means clustering 기반으로 대표 RGB와 색상 비율을 추출한다.

### 처리 흐름

```text
이미지 입력
→ 이미지 전처리
→ 픽셀 RGB 데이터 추출
→ K-means clustering 실행
→ 대표 색상 후보 생성
→ 색상 비율 계산
→ 색상 기준 테이블과 매핑
→ 분석 결과 저장
```

### 색상 추출 결과

- 대표 RGB
- 색상명
- 색상 비율
- 클러스터 순서
- 대표 색상 여부
- 색상 성격 문구

### 활용 기능

- 작품 DB 생성
- 공간 이미지 기반 작품 추천
- 모바일 리플릿 대표 색상 추출
- 색상 성격 문구 생성
- 전시 추천 보조

---

## 3. 감성 매핑 단계

감성 매핑은 작품의 분위기와 사용자의 취향을 연결하기 위한 단계이다.

작품 이미지, 작품명, 작품 설명을 기반으로 AI가 감성을 추출하고, 이를 RestArt 자체 감성 카테고리에 매핑한다.

### 처리 흐름

```text
작품 이미지
+ 작품명
+ 작품 설명
→ 멀티모달 감성 분석
→ 감성 후보 추출
→ RestArt 감성 카테고리와 매핑
→ 상위 감성 태그 선정
→ 작가 또는 운영자 검수
→ 작품 감성 DB 저장
```

### 감성 매핑 결과

- 감성명
- 감성 카테고리
- 감성 순위
- AI 추출 여부
- 작가 수정 여부
- 운영자 보정 여부

### 활용 기능

- 작품 추천 정렬
- 사용자 취향 분석
- 리플릿 추천 작품 선정
- 추천 사유 생성

## 4. 공간 이미지 기반 작품 추천

공간 기반 추천은 사용자가 작품을 배치하고 싶은 실내 사진을 업로드하는 기능이다.

시스템은 공간 이미지에서 색상을 추출하고, 전문가 인테리어 DB 및 작품 DB와 비교하여 적합한 작품을 추천한다.

### 처리 흐름

```text
사용자 공간 이미지 업로드
→ 공간 이미지 색상 추출
→ 전문가 인테리어 DB와 비교
→ 유사 인테리어 탐색
→ 전문가 기준 작품 탐색
→ RestArt 작품 DB와 비교
→ 감성 일치 개수 계산
→ 켄달타우 점수 계산
→ 추천 작품 정렬
→ 추천 결과 저장
```

### 추천 기준

1. 감성 일치 개수
2. 켄달타우 점수
3. 색상 유사도
4. 공간 분위기와 작품 분위기의 조화

### 연결 DB

- `user_space_images`
- `expert_interiors`
- `expert_artworks`
- `artworks`
- `artwork_colors`
- `artwork_emotions`
- `recommendation_results`

## 5. 유사도 및 순위 계산

RestArt의 추천 정렬은 단순 색상 유사도만 사용하지 않는다.

색상 기반 비교와 감성 기반 비교를 함께 사용한다.

### 기본 정렬 기준

```text
1순위: 감성 일치 개수
2순위: 켄달타우 점수
```

감성이 더 많이 일치하는 작품을 우선 추천하고, 감성 일치 개수가 같은 경우 켄달타우 점수가 높은 작품을 우선 배치한다.

### 켄달타우 활용 목적

켄달타우는 색상 비율 또는 색상 순위 간의 순서 유사성을 비교하는 데 사용한다.

예를 들어 사용자 공간 이미지에서 추출된 주요 색상 순위와 작품 이미지에서 추출된 주요 색상 순위가 비슷하면, 두 이미지의 색상 구성 흐름이 유사하다고 판단할 수 있다.

## 6. 모바일 리플릿 추천

모바일 리플릿 기능은 사용자가 전시장에서 촬영한 작품 이미지를 기반으로 한다.

촬영된 작품 이미지는 DB의 작품 데이터와 매핑되고, 사용자의 촬영 이미지 묶음에서 대표 색상과 대표 작품을 추출한다.

### 처리 흐름

```text
전시장 작품 촬영
→ 촬영 이미지 저장
→ 전시 작품 DB 또는 작품 DB와 이미지 매칭
→ 리플릿 세션 생성
→ 촬영 작품들의 대표 색상 계산
→ 대표 작품 선정
→ 색상 성격 문구 생성
→ 추천 작품 생성
→ 추천 전시 생성
→ 모바일 리플릿 결과 저장
```

### 리플릿 결과 구성

- 대표 색상
- 색상 기반 성격 문구
- 사용자가 촬영한 작품 이미지
- 대표 작품
- 추천 작품
- 추천 전시
- SNS 공유용 결과

### 연결 DB

- `leaflet_sessions`
- `leaflet_images`
- `leaflet_recommendations`
- `artworks`
- `exhibitions`
- `color_dictionary`

## 9. 추천 시스템의 특징

RestArt AI 추천 시스템의 특징은 다음과 같다.

1. 색상과 감성을 함께 활용한다.
2. 작품 DB를 중심으로 추천 기능과 리플릿 기능을 연결한다.
3. 사용자 공간 이미지와 전시 촬영 이미지를 모두 추천 입력값으로 사용한다.
4. K-means 기반 RGB 추출을 통해 이미지의 시각적 특징을 정량화한다.
5. RestArt 감성 테이블을 통해 작품의 정서적 특징을 구조화한다.
6. 켄달타우 점수를 활용하여 색상 순위 기반 유사성을 계산한다.
7. 추천 결과를 DB에 저장하여 재조회와 개선이 가능하도록 설계한다.

## 향후 개선 방향

향후 추천 시스템은 다음 방향으로 개선할 수 있다.

- 추천 점수 계산식 세분화
- 사용자 피드백 기반 추천 품질 개선
- 작품 클릭/저장/구매 로그 반영
- 작가 선호도 기반 추천 추가
- 전시 위치 기반 추천 고도화
- AR 배치 결과와 추천 결과 연동
- 색상·감성·공간 스타일 통합 임베딩 모델 도입
