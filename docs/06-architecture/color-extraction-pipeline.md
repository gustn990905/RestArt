# Color Extraction Pipeline

## 개요

이 문서는 RestArt 프로젝트의 색상 추출 파이프라인을 정리한다.

RestArt는 작품 이미지, 사용자 공간 이미지, 전시 촬영 이미지에서 RGB 색상값과 색상 비율을 추출하고, 이를 작품 추천과 모바일 리플릿 생성에 활용한다.

색상 추출은 RestArt 추천 시스템의 핵심 기반 기능이다.

---

## 색상 추출의 목적

색상 추출의 목적은 다음과 같다.

1. 작품 이미지의 대표 색상 추출
2. 사용자 공간 이미지의 주요 색상 분석
3. 작품과 공간 이미지의 색상 유사도 비교
4. 모바일 리플릿의 대표 색상 생성
5. 색상 기반 성격 문구 생성
6. 켄달타우 기반 색상 순위 비교에 필요한 입력값 생성

---

## 입력 데이터

색상 추출 파이프라인의 입력 데이터는 다음과 같다.

| 입력 데이터      | 설명                                   | 활용 기능                        |
| ---------------- | -------------------------------------- | -------------------------------- |
| 작품 이미지      | 작가가 등록한 작품 이미지              | 작품 DB 생성, 추천 후보 분석     |
| 공간 이미지      | 사용자가 업로드한 실내 이미지          | 공간 기반 작품 추천              |
| 전시 촬영 이미지 | 사용자가 전시장에서 촬영한 작품 이미지 | 모바일 리플릿 생성               |
| 기준 색상 테이블 | RestArt 색상명, RGB, 톤 정보           | 색상명 매핑, 색상 성격 문구 생성 |

---

## 전체 처리 흐름

```text
이미지 입력
→ 이미지 전처리
→ RGB 픽셀 데이터 추출
→ K-means clustering 실행
→ 대표 색상 후보 추출
→ 색상 비율 계산
→ 대표 색상 선정
→ 색상명 매핑
→ DB 저장 또는 추천 알고리즘 입력값으로 전달
```

---

## 1. 이미지 입력

색상 추출은 이미지 파일 또는 이미지 URL을 입력으로 받는다.

이미지는 다음 경로에서 들어올 수 있다.

- 작가 작품 등록 이미지
- 사용자 공간 업로드 이미지
- 전시장에서 촬영한 작품 이미지
- 추천 후보 작품 이미지
- 전문가 인테리어 이미지
- 전문가 기준 작품 이미지

---

## 2. 이미지 전처리

입력 이미지는 색상 추출 전에 기본 전처리를 수행한다.

### 전처리 항목

- 이미지 파일 로드
- RGB 모드 변환
- 이미지 크기 조정
- 불필요한 투명도 채널 제거
- 픽셀 배열 변환
- 너무 어둡거나 너무 밝은 픽셀 필터링 검토

### 목적

이미지마다 해상도와 색상 모드가 다를 수 있으므로, K-means clustering을 적용하기 전에 분석 가능한 RGB 배열로 통일한다.

---

## 3. RGB 픽셀 데이터 추출

전처리된 이미지를 RGB 픽셀 배열로 변환한다.

각 픽셀은 다음과 같은 형태를 가진다.

```text
[R, G, B]
```

예시:

```text
[59, 130, 157]
[227, 189, 28]
[126, 126, 126]
```

이 RGB 배열은 K-means clustering의 입력값으로 사용된다.

---

## 4. K-means clustering

RestArt는 이미지의 주요 색상을 추출하기 위해 K-means clustering을 사용한다.

K-means는 이미지 안의 RGB 픽셀들을 색상 유사도 기준으로 여러 그룹으로 나누고, 각 그룹의 중심값을 대표 색상으로 사용한다.

### 기본 설정

```text
k = 10
```

즉, 하나의 이미지에서 최대 10개의 색상 클러스터를 추출한다.

### 처리 결과

K-means clustering 결과는 다음과 같다.

- 클러스터 중심 RGB
- 각 클러스터에 포함된 픽셀 수
- 전체 이미지 대비 클러스터 비율
- 색상 순위
- 대표 색상 후보

---

## 5. 색상 비율 계산

각 클러스터의 픽셀 개수를 전체 픽셀 개수로 나누어 색상 비율을 계산한다.

```text
색상 비율 = 해당 클러스터 픽셀 수 / 전체 픽셀 수
```

예시:

| 색상 순서 | RGB               | 비율 |
| --------- | ----------------- | ---- |
| 1         | `[59, 130, 157]`  | 0.35 |
| 2         | `[126, 126, 126]` | 0.18 |
| 3         | `[227, 189, 28]`  | 0.12 |

색상 비율은 작품 이미지와 공간 이미지의 색상 구성 비교에 사용된다.

---

## 6. 대표 색상 선정

대표 색상은 일반적으로 가장 높은 비율을 가진 클러스터를 기준으로 선정한다.

다만 이미지의 배경색이 과도하게 큰 비중을 차지할 경우, 실제 작품의 인상과 대표색이 다를 수 있으므로 향후 다음 기준을 추가할 수 있다.

- 배경색 제외
- 무채색 비율 보정
- 채도 기준 보정
- 명도 기준 보정
- 작품 영역 중심 색상 우선 반영

현재 단계에서는 포트폴리오 문서화 기준으로 K-means 결과의 상위 색상을 대표 색상 후보로 사용한다.

---

## 7. 색상명 매핑

추출된 RGB 값은 RestArt 색상 기준 테이블과 매핑된다.

매핑 기준은 다음과 같다.

1. 추출된 RGB 값과 기준 색상 RGB 값 비교
2. RGB 거리 계산
3. 가장 가까운 기준 색상 선택
4. 색상명과 톤 그룹 부여
5. 리플릿용 색상 성격 문구 연결

### RGB 거리 예시

```text
distance = sqrt((R1 - R2)^2 + (G1 - G2)^2 + (B1 - B2)^2)
```

RGB 거리가 가장 작은 기준 색상을 해당 색상의 이름으로 매핑한다.

---

## 8. 저장 데이터 구조

작품 이미지에서 추출된 색상은 `artwork_colors`에 저장된다.

사용자 공간 이미지의 색상 분석 결과는 `user_space_images`에 저장된다.

리플릿 대표 색상은 `leaflet_sessions`에 저장된다.

### 작품 색상 저장 예시

| 필드            | 설명           |
| --------------- | -------------- |
| `artwork_id`    | 작품 ID        |
| `color_name`    | 매핑된 색상명  |
| `rgb_r`         | R 값           |
| `rgb_g`         | G 값           |
| `rgb_b`         | B 값           |
| `ratio`         | 색상 비율      |
| `cluster_count` | 클러스터 count |
| `cluster_order` | 색상 순위      |
| `is_main_color` | 대표 색상 여부 |

---

## 9. 추천 알고리즘과의 연결

색상 추출 결과는 추천 알고리즘의 입력값으로 사용된다.

### 공간 기반 작품 추천

```text
사용자 공간 이미지 색상
→ 전문가 인테리어 색상과 비교
→ 기준 작품 탐색
→ RestArt 작품 DB와 색상 순위 비교
→ 켄달타우 점수 계산
```

### 모바일 리플릿 추천

```text
전시 촬영 이미지 색상
→ 대표 RGB 계산
→ 색상 성격 문구 생성
→ 색상 기반 추천 작품 탐색
→ 리플릿 결과 생성
```

---

## 10. 켄달타우 계산과 색상 순위

RestArt에서는 색상 유사도를 단순 RGB 거리만으로 판단하지 않는다.

이미지에서 추출된 색상들의 순서와 비율을 함께 고려한다.

예를 들어 공간 이미지의 주요 색상 순서가 다음과 같다고 가정한다.

```text
공간 이미지: Blue → Grey → Yellow
```

작품 이미지의 주요 색상 순서가 다음과 같다면:

```text
작품 A: Blue → Grey → Yellow
작품 B: Yellow → Red → Green
```

작품 A는 색상 순서가 공간 이미지와 유사하므로 더 높은 점수를 받을 수 있다.

이때 색상 순위 간 유사성을 계산하는 방식으로 켄달타우를 활용한다.

---

## 11. 출력 데이터

색상 추출 파이프라인의 최종 출력은 다음과 같다.

| 출력값           | 설명                                    |
| ---------------- | --------------------------------------- |
| `dominant_rgb`   | 대표 RGB                                |
| `main_color`     | 대표 색상명                             |
| `color_clusters` | 전체 색상 클러스터 목록                 |
| `color_ratios`   | 각 색상 비율                            |
| `cluster_order`  | 색상 순위                               |
| `color_summary`  | 추천 알고리즘에 전달할 색상 요약 데이터 |

---

## 향후 개선 방향

향후 색상 추출 파이프라인은 다음 방향으로 개선할 수 있다.

- 배경색 제거 로직 추가
- 작품 영역 segmentation 적용
- 조명 보정 로직 추가
- 무채색 과대표현 보정
- 색상명 매핑 정확도 개선
- HSV, LAB 색공간 비교 추가
- 사용자 선호 색상 피드백 반영
