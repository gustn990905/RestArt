# Emotion Mapping Pipeline

## 개요

이 문서는 RestArt 프로젝트의 감성 매핑 파이프라인을 정리한다.

RestArt는 작품 이미지, 작품명, 작품 설명을 기반으로 작품의 감성을 추출하고, 이를 RestArt 자체 감성 카테고리에 매핑한다.

감성 매핑 결과는 공간 기반 작품 추천, 모바일 리플릿 추천, 작품 검색, 추천 사유 생성에 활용된다.

---

## 감성 매핑의 목적

감성 매핑의 목적은 다음과 같다.

1. 작품의 정서적 특징을 구조화한다.
2. 작품 이미지와 설명을 감성 데이터로 변환한다.
3. 사용자 공간 분위기와 작품 감성을 연결한다.
4. 리플릿 추천 작품 선정에 활용한다.
5. 추천 결과에 설명 가능한 이유를 제공한다.
6. 작가가 의도한 감성과 AI가 추출한 감성을 함께 관리한다.

---

## 입력 데이터

감성 매핑 파이프라인의 입력 데이터는 다음과 같다.

| 입력 데이터              | 설명                            |
| ------------------------ | ------------------------------- |
| 작품 이미지              | 작품의 시각적 분위기 분석       |
| 작품명                   | 작품 제목에 포함된 정서적 단서  |
| 작품 설명                | 작가가 작성한 작품 해석 및 의도 |
| 작가 수정 감성           | 작가가 직접 보정한 감성 태그    |
| RestArt 감성 기준 테이블 | 감성명, 감성 카테고리, 설명     |

---

## 전체 처리 흐름

```text
작품 이미지 입력
+ 작품명 입력
+ 작품 설명 입력
→ 멀티모달 감성 분석
→ 감성 후보 추출
→ RestArt 감성 기준 테이블과 매핑
→ 상위 감성 태그 선정
→ 작가 검수 및 수정
→ 작품 감성 DB 저장
→ 추천 알고리즘 입력값으로 사용
```

---

## 1. 작품 데이터 입력

작가가 작품을 등록할 때 다음 데이터를 입력한다.

- 작품 이미지
- 작품명
- 작품 설명
- 작가명 또는 닉네임
- 작품 가격
- 작품 상태

이 중 감성 매핑에 직접 사용되는 데이터는 다음이다.

```text
작품 이미지
작품명
작품 설명
```

---

## 2. 멀티모달 감성 분석

감성 분석은 단순 텍스트 분석만으로 수행하지 않는다.

작품은 이미지 기반 콘텐츠이기 때문에 작품 이미지와 텍스트 설명을 함께 고려하는 멀티모달 방식이 적합하다.

### 분석 기준

| 데이터      | 분석 역할                            |
| ----------- | ------------------------------------ |
| 작품 이미지 | 색감, 구도, 분위기, 시각적 인상 분석 |
| 작품명      | 작가가 부여한 정서적 방향성 분석     |
| 작품 설명   | 작품 의도, 서사, 상징성 분석         |

---

## 3. 감성 후보 추출

AI는 작품 데이터에서 감성 후보를 추출한다.

예시 감성 후보는 다음과 같다.

```text
행복
평화
호기심
감각적
우울
긴장
따뜻함
차분함
그리움
```

감성 후보는 그대로 저장하지 않고 RestArt 감성 기준 테이블에 맞게 정리한다.

---

## 4. RestArt 감성 기준 테이블 매핑

RestArt는 자체 감성 기준 테이블을 사용한다.

감성 후보는 이 기준 테이블의 감성명 또는 감성 카테고리와 매핑된다.

### 매핑 목적

- 작품 감성 표현을 통일한다.
- 추천 알고리즘에서 비교 가능한 데이터로 만든다.
- 작품별 감성 태그를 구조화한다.
- 감성 일치 개수를 계산할 수 있게 한다.

### 저장 대상

- `emotion_dictionary`
- `artwork_emotions`

---

## 5. 상위 감성 태그 선정

하나의 작품에는 여러 감성이 포함될 수 있다.

RestArt는 추천 알고리즘에서 사용하기 위해 작품별 대표 감성을 상위 몇 개로 정리한다.

기본 구조는 다음과 같다.

```text
emotion_1
emotion_2
emotion_3
```

또는 DB에서는 다음과 같이 저장한다.

| 필드           | 설명                               |
| -------------- | ---------------------------------- |
| `emotion_name` | 감성명                             |
| `emotion_rank` | 감성 순위                          |
| `source`       | AI, 작가, 운영자 중 감성 생성 출처 |

---

## 6. 작가 검수 및 수정

AI가 추출한 감성은 작가가 수정할 수 있다.

이 단계는 RestArt 감성 데이터 품질을 높이는 중요한 과정이다.

### 수정이 필요한 이유

- AI가 작품 의도를 완전히 해석하지 못할 수 있음
- 작가가 의도한 감성과 관람자가 느끼는 감성이 다를 수 있음
- 작품 설명이 부족한 경우 이미지 기반 분석이 과도하게 반영될 수 있음
- 추천 품질을 높이기 위해 작가 의도를 반영할 필요가 있음

### 감성 출처 구분

| source   | 의미                               |
| -------- | ---------------------------------- |
| `ai`     | AI가 자동 추출한 감성              |
| `artist` | 작가가 수정하거나 직접 입력한 감성 |
| `admin`  | 운영자가 보정한 감성               |

---

## 7. DB 저장 구조

감성 매핑 결과는 `artwork_emotions`에 저장된다.

기준 감성 목록은 `emotion_dictionary`에 저장된다.

### `artwork_emotions`

| 필드           | 설명           |
| -------------- | -------------- |
| `artwork_id`   | 작품 ID        |
| `emotion_id`   | 기준 감성 ID   |
| `emotion_name` | 감성명         |
| `emotion_rank` | 감성 순위      |
| `source`       | 감성 생성 출처 |
| `created_at`   | 생성일         |

### `emotion_dictionary`

| 필드               | 설명                |
| ------------------ | ------------------- |
| `emotion_id`       | 감성 ID             |
| `emotion_name`     | 감성명              |
| `emotion_category` | 감성 분류           |
| `description`      | 감성 설명           |
| `source_reference` | 감성 정의 참고 기준 |

---

## 8. 추천 알고리즘과의 연결

감성 매핑 결과는 작품 추천 정렬에 직접 사용된다.

RestArt의 기본 추천 정렬 기준은 다음과 같다.

```text
1순위: 감성 일치 개수
2순위: 켄달타우 점수
```

즉, 색상 유사도가 높더라도 감성 일치가 낮으면 우선순위가 낮아질 수 있다.

### 감성 일치 개수 예시

사용자 공간 또는 기준 작품 감성:

```text
행복, 평화, 감각적
```

후보 작품 A:

```text
행복, 평화, 호기심
```

일치 감성:

```text
행복, 평화
```

감성 일치 개수:

```text
2개
```

이 값은 `recommendation_results.emotion_match_count`에 저장될 수 있다.

---

## 9. 모바일 리플릿과의 연결

모바일 리플릿에서는 사용자가 전시장에서 촬영한 작품 이미지들이 취향 데이터가 된다.

촬영 작품이 DB 작품과 매핑되면, 해당 작품들의 감성 태그를 기반으로 사용자의 전시 취향을 해석할 수 있다.

### 활용 방식

- 촬영 작품들의 공통 감성 추출
- 사용자의 선호 감성 추정
- 유사 감성 작품 추천
- 전시 추천 보조
- 리플릿 설명 문구 생성

---

## 10. 출력 데이터

감성 매핑 파이프라인의 최종 출력은 다음과 같다.

| 출력값                | 설명                                          |
| --------------------- | --------------------------------------------- |
| `emotion_tags`        | 작품별 감성 태그 목록                         |
| `emotion_rank`        | 감성 우선순위                                 |
| `emotion_source`      | AI, 작가, 운영자 구분                         |
| `emotion_match_count` | 추천 정렬에 사용되는 감성 일치 개수           |
| `emotion_summary`     | 추천 사유 또는 리플릿 문구에 사용할 감성 요약 |

---

## 향후 개선 방향

향후 감성 매핑 파이프라인은 다음 방향으로 개선할 수 있다.

- 감성 카테고리별 신뢰도 점수 저장
- 작가 수정 이력 관리
- 사용자 반응 기반 감성 보정
- 작품 설명이 없는 경우 이미지 기반 fallback 강화
- 감성 태그 자동 추천 UI 추가
- 감성 기반 검색 필터 구현
- 추천 사유 자동 생성 고도화
