# Similarity Ranking Logic

## 개요

이 문서는 RestArt의 작품 추천 정렬 로직을 정리한다.

RestArt의 추천 시스템은 단순히 색상이 비슷한 작품을 추천하는 방식이 아니라, 색상 구성과 감성 태그를 함께 비교하여 추천 순위를 결정한다.

추천 정렬의 핵심 기준은 다음 두 가지이다.

```text
1순위: 감성 일치 개수
2순위: 켄달타우 점수
```

즉, 먼저 감성이 더 많이 일치하는 작품을 우선 추천하고, 감성 일치 개수가 같은 작품들 사이에서는 색상 순위 유사도를 기준으로 정렬한다.

---

## 추천 정렬의 목적

추천 정렬 로직의 목적은 다음과 같다.

1. 사용자 공간과 어울리는 작품을 우선 추천한다.
2. 작품의 감성적 분위기를 추천 기준에 반영한다.
3. 이미지의 색상 순서와 비율을 비교한다.
4. 감성 일치와 색상 유사도를 함께 고려한다.
5. 추천 결과에 설명 가능한 기준을 부여한다.

---

## 입력 데이터

추천 정렬 로직은 다음 데이터를 입력값으로 사용한다.

| 입력 데이터             | 설명                                                      |
| ----------------------- | --------------------------------------------------------- |
| 사용자 공간 이미지 색상 | 사용자가 업로드한 공간 이미지에서 추출된 RGB 및 색상 비율 |
| 기준 작품 색상          | 전문가 기준 작품 또는 비교 기준 작품의 색상 정보          |
| 후보 작품 색상          | RestArt 작품 DB에 저장된 작품별 색상 정보                 |
| 기준 감성 태그          | 공간 또는 기준 작품과 연결된 감성 정보                    |
| 후보 작품 감성 태그     | 추천 후보 작품의 감성 정보                                |
| 색상 순위               | K-means 결과로 추출된 주요 색상 순서                      |
| 색상 비율               | 이미지 내 각 색상의 비중                                  |

---

## 전체 처리 흐름

```text
사용자 공간 이미지 업로드
→ 색상 추출
→ 전문가 인테리어 DB와 비교
→ 기준 작품 탐색
→ 후보 작품 목록 조회
→ 감성 일치 개수 계산
→ 켄달타우 점수 계산
→ 추천 점수 정렬
→ 추천 결과 저장
```

---

## 1. 후보 작품 목록 생성

추천 정렬을 수행하기 전에 먼저 추천 후보 작품 목록을 생성한다.

후보 작품은 다음 데이터에서 가져올 수 있다.

- RestArt 참여 작가 작품
- 추천 후보 작품 DB
- 전시 기반 작품 DB
- 전문가 기준 작품과 유사한 작품

DB 기준으로는 `artworks` 테이블의 작품들이 후보군이 된다.

작품의 색상 정보는 `artwork_colors`에서 조회하고, 작품의 감성 정보는 `artwork_emotions`에서 조회한다.

---

## 2. 감성 일치 개수 계산

RestArt 추천 정렬에서 가장 먼저 적용되는 기준은 감성 일치 개수이다.

감성 일치 개수는 기준 감성 태그와 후보 작품 감성 태그가 얼마나 많이 겹치는지를 계산한 값이다.

### 예시

기준 감성:

```text
행복, 평화, 감각적
```

후보 작품 A:

```text
행복, 평화, 호기심
```

일치 감성:

```text
행복, 평화
```

감성 일치 개수:

```text
2
```

후보 작품 B:

```text
긴장, 우울, 고독
```

일치 감성:

```text
없음
```

감성 일치 개수:

```text
0
```

이 경우 후보 작품 A가 후보 작품 B보다 우선 추천된다.

---

## 3. 감성 일치 개수를 1순위로 사용하는 이유

작품 추천은 색상이 비슷하다고 항상 적절한 것은 아니다.

예를 들어 공간 색상은 비슷하더라도, 작품의 감성이 공간 분위기와 맞지 않으면 추천 만족도가 떨어질 수 있다.

따라서 RestArt는 색상 유사도보다 감성 일치 개수를 먼저 적용한다.

### 기준

```text
감성 일치 개수가 높은 작품
→ 사용자 공간 또는 취향과 정서적으로 더 잘 맞는 작품으로 판단
```

---

## 4. 켄달타우 점수 계산

감성 일치 개수가 같은 작품들 사이에서는 켄달타우 점수를 기준으로 정렬한다.

켄달타우는 두 순위 목록의 순서 유사성을 비교하는 통계적 방법이다.

RestArt에서는 이미지에서 추출된 주요 색상 순서가 얼마나 유사한지를 비교하는 데 사용한다.

---

## 5. 색상 순위 비교 방식

K-means clustering 결과로 각 이미지의 주요 색상 순위를 얻는다.

예시:

```text
사용자 공간 이미지:
1. Blue
2. Grey
3. Yellow
4. White
```

후보 작품 A:

```text
1. Blue
2. Grey
3. Yellow
4. Black
```

후보 작품 B:

```text
1. Red
2. Yellow
3. Green
4. Blue
```

후보 작품 A는 사용자 공간 이미지와 색상 순서가 더 유사하다.

따라서 후보 작품 A의 켄달타우 점수가 더 높게 계산된다.

---

## 6. 켄달타우를 사용하는 이유

단순 RGB 거리만 비교하면 색상의 전체 구성 순서를 반영하기 어렵다.

예를 들어 두 이미지가 같은 색상을 포함하고 있어도, 어떤 색상이 주된 색상인지에 따라 시각적 인상이 달라질 수 있다.

켄달타우는 다음 차이를 반영할 수 있다.

- 주요 색상 순서
- 색상 비중의 상대적 순위
- 공간 이미지와 작품 이미지의 색상 구성 유사성

---

## 7. 추천 정렬 기준

RestArt의 기본 추천 정렬 기준은 다음과 같다.

```text
1. 감성 일치 개수 내림차순
2. 켄달타우 점수 내림차순
3. 색상 비율 유사도 내림차순
4. 최신 등록 작품 또는 운영 정책 기준
```

현재 포트폴리오 기준에서는 1번과 2번을 핵심 정렬 기준으로 문서화한다.

---

## 8. 추천 결과 저장 구조

추천 결과는 `recommendation_results`에 저장된다.

| 필드                    | 설명                          |
| ----------------------- | ----------------------------- |
| `user_id`               | 추천을 요청한 사용자          |
| `space_image_id`        | 사용자가 업로드한 공간 이미지 |
| `artwork_id`            | 추천된 작품                   |
| `emotion_match_count`   | 감성 일치 개수                |
| `kendall_tau_score`     | 켄달타우 점수                 |
| `rank_order`            | 추천 순위                     |
| `recommendation_reason` | 추천 사유                     |
| `created_at`            | 추천 생성일                   |

---

## 9. 추천 사유 생성

추천 결과에는 단순히 작품 목록만 제공하지 않고, 추천 사유를 함께 제공할 수 있다.

예시:

```text
이 작품은 사용자의 공간 이미지와 주요 색상 구성이 유사하고,
행복, 평화 감성이 함께 일치하여 추천되었습니다.
```

추천 사유는 다음 데이터를 바탕으로 생성할 수 있다.

- 일치한 감성명
- 대표 색상명
- 색상 순위 유사도
- 공간 이미지의 대표 색상
- 작품의 대표 색상

---

## 10. 공간 기반 추천에서의 적용

공간 기반 작품 추천에서는 다음 방식으로 정렬 로직을 적용한다.

```text
사용자 공간 이미지
→ 공간 색상 추출
→ 전문가 인테리어 DB 비교
→ 기준 작품 추출
→ 후보 작품 DB 비교
→ 감성 일치 개수 계산
→ 켄달타우 점수 계산
→ 추천 순위 결정
```

이 결과는 사용자에게 추천 작품 목록으로 제공된다.

---

## 11. 모바일 리플릿 추천에서의 적용

모바일 리플릿에서는 사용자가 촬영한 작품들이 사용자의 취향 데이터가 된다.

촬영 작품에서 추출된 색상과 감성 정보를 기반으로 추천 후보 작품을 정렬한다.

```text
촬영 작품 목록
→ 대표 색상 및 감성 추출
→ 후보 작품 DB 비교
→ 감성 일치 개수 계산
→ 켄달타우 점수 계산
→ 리플릿 추천 작품 선정
```

---

## 12. 향후 개선 방향

향후 추천 정렬 로직은 다음 방향으로 개선할 수 있다.

- 켄달타우 점수 외 색상 거리 점수 추가
- 감성별 가중치 적용
- 작가 선호도 반영
- 사용자 저장/클릭/구매 로그 반영
- 전시 위치 기반 가중치 반영
- 추천 결과 피드백 기반 재정렬
- 설명 가능한 추천 문구 자동 생성 고도화

---

## 정리

RestArt의 추천 정렬 로직은 감성과 색상을 함께 고려한다.

감성 일치 개수는 작품의 분위기 적합성을 판단하는 1차 기준이고, 켄달타우 점수는 이미지 색상 순위의 유사성을 판단하는 2차 기준이다.

이 구조를 통해 RestArt는 단순 색상 매칭을 넘어, 공간 분위기와 작품 감성을 함께 반영하는 추천 시스템을 구현한다.
